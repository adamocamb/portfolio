Machined Aluminum Rear Bike Rack

A bicycle-specific design project intended to show the full loop: requirements, CAD, analysis, manufacturing planning, drawings, prototype build, test plan, and iteration.

Snapshot

Status
Design/modeling started; build not complete yet
Why it matters
Public, bike-specific, SRAM-adjacent evidence of mechanical design process
Tools to use
Inventor or SolidWorks, Inventor Nastran, shop tools, GD&T drawing package
Best outcome
A documented design review, not just a finished accessory

Project Goal

Design and build a machined aluminum rear rack that fits a real bike, clears the tire and fender envelope, supports practical cargo loads, is serviceable with normal tools, and can be manufactured with available shop processes.

Recommended Requirements

Requirement Portfolio evidence
Load capacity and stiffness target Hand calculation, FEA load cases, static load test
Frame and axle/fender/tire clearance Interface drawing, CAD screenshots, tolerance stack
Manufacturable geometry DFM notes, operation plan, setup sketches, drawing packet
Fastener retention and serviceability Torque spec logic, tool-access screenshots, inspection plan
Durability under real use Static overload test, road ride log, post-test inspection photos

Analysis Plan

Use this as a deliberate chance to ramp Inventor Nastran and show analysis discipline without pretending the model is more accurate than it is.

  • Start with free-body diagrams and beam/plate hand calculations for expected stress paths.
  • Run three simple FEA cases: vertical cargo load, side load, and asymmetric pannier load.
  • Show boundary-condition assumptions clearly, especially mounting constraints and load introduction.
  • Compare hand calculations, FEA trends, and physical test results at a few inspection points.
  • Use the result to change geometry, not just to make a colorful plot.

Manufacturing Plan

Prototype fit check

3D print or cut simple templates to confirm tire, fender, heel, and tool clearances before machining aluminum.

Machined components

Design plates, spacers, and struts around available stock, cutter access, fixturing, deburring, and inspection.

Drawing package

Create concise manufacturing drawings with datum logic, critical dimensions, GD&T where useful, and inspection notes.

Validation

Build a simple test fixture or loading method, document before/after inspection, and record any design changes.

Why This Should Be Built Next

For the SRAM application, this is more valuable than another generic small build. It is bicycle hardware, it can be photographed and explained publicly, it naturally involves mounting interfaces and tolerance decisions, and it gives you a reason to show analysis, shop process, drawings, and validation in a single clean story.
